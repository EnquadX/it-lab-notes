# Google IT Support — Module 1: Introduction to IT

These are my takeaways from the first module of the Coursera Google IT Support course.

I wanted this note to be short enough to review later, but still useful enough that I actually remember what I learned.

## Big picture: how computing works

The simplest way I understand it now is in 4 layers:

- Users
- Software
- Operating system
- Hardware

A useful memory shortcut:
- **U-S-O-H**
- Users → Software → OS → Hardware

The main idea here is **abstraction**.
That means complexity gets hidden behind simpler layers and interfaces, so you can use something without needing to understand every lower layer all the time.

## Core definitions worth remembering

A few terms from this module feel like baseline IT vocabulary:

- **Computer** — stores and processes data by performing calculations
- **Information technology (IT)** — using digital technology to turn data into useful information
- **Abstraction** — simplifying a more complex system for practical use
- **Algorithm** — a series of steps that solves a specific problem
- **Open source** — software other people can inspect, modify, and distribute
- **Linux** — a major open-source operating system used heavily in business and infrastructure
- **Digital divide** — the skills gap between people with and without digital literacy
- **Cryptography** — the broader field of encoding, protecting, and hiding information

## A few useful history points

The course also helped connect a few older computing ideas together:

- punch cards were used to automate calculations
- magnetic tape became an important storage step after that
- vacuum tubes were replaced by transistors
- transistors mattered because they were smaller, faster, and more efficient
- MS-DOS was part of the command-line desktop era
- PDAs helped move computing toward mobile devices

A simple upgrade chain that stuck with me:

- **Punch cards → tape**
- **Vacuum tubes → transistors**

## Binary basics

This module also reinforced the idea that computers operate in **binary**.

Main points:

- decimal = base 10
- binary = base 2
- a **bit** = 1 binary digit
- a **byte** = 8 bits
- 1 byte = 256 combinations

It also covered how binary becomes readable text through **character encoding**.

Two useful names:

- **ASCII** — older standard for English letters, digits, punctuation
- **UTF-8** — modern common standard that supports much more, including symbols and emojis

## Logic gates

This was one of the more useful foundational parts of the module.

The basic idea:

- input comes in as 0 or 1
- a logic rule gets applied
- output comes out as 0 or 1

The six gates I want to remember:

- NOT
- AND
- OR
- XOR
- NAND
- XNOR

The two quick formulas worth keeping in mind:

- **NAND = NOT(AND)**
- **XNOR = NOT(XOR)**

The main takeaway is that computers are built from very large numbers of these logic operations chained together.
